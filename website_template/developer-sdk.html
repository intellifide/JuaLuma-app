Available Scopes

Widgets can request read-only access to specific data types. All scopes are reviewed during security review.

‚úì Allowed Scopes

  • ledger:read - Read transaction ledger
  • accounts:read - Read account metadata
  • categories:read - Read category definitions
  • budgets:read - Read budget limits and spending
  • analytics:read - Read aggregated analytics
  • recurring:read - Read recurring transaction patterns

‚úó Prohibited

  • No write operations - Cannot modify any data
  • No transfers - Cannot initiate money movements
  • No trades - Cannot execute investment trades
  • No secrets - Cannot access API keys or credentials
  • No PII - Limited to necessary data only
Scope Validation: Requesting disallowed scopes will cause your submission to be rejected. Only request scopes your widget actually needs.

Installation & CLI

Install SDK

npm install @jualuma/sdk

Initialize Widget

# Initialize new widget project jualuma dev init my-widget # This creates: # - widget.json (manifest) # - src/index.js (widget entry point) # - preview-data.json (synthetic data for preview mode) # - .jualumaignore (files to exclude from bundle)

Example Widget Code

import { jualumaSDK } from '@jualuma/sdk'; const sdk = new jualumaSDK({ scopes: ['ledger:read', 'budgets:read'], previewEnabled: true }); // Listen for transaction events sdk.on('transaction', (transaction) => { console.log('New transaction:', transaction); updateBudgetDisplay(transaction); }); // Access ledger data const transactions = await sdk.ledger.getTransactions({ startDate: '2025-01-01', endDate: '2025-01-31' }); // Access budget data const budgets = await sdk.budgets.getAll();

Build & Validate

# Build widget bundle jualuma dev build # Validate manifest and scopes jualuma dev validate # Run in local sandbox jualuma dev sandbox
Note: This is a mock SDK interface. In production, the CLI will provide full validation, bundling, and local testing capabilities.
